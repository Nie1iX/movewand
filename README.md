# MoveWand

MoveWand — Fabric-мод для Minecraft с выживальным инструментом, который позволяет аккуратно перемещать и поворачивать уже построенные блоки.

Это не замена WorldEdit и не система копирования построек. Жезл нужен, когда надо исправить расположение механизма, освободить место или повернуть часть конструкции без ручного демонтажа и сборки заново.

## Принципы

- Доступный крафт: палка, два железных слитка и алмаз.
- Удобство без искусственной цены: операции не тратят голод и прочность.
- Безопасность важнее магии: действие отменяется, если новое положение нельзя занять без потери блоков.
- В multiplayer мир изменяет только сервер; клиент отвечает за выделение, обводку и управление.
- Жёсткие пределы по размеру выделения и дистанции защищают мир и сервер от случайных массовых операций: 512 блоков и 16 блоков соответственно.

## Текущее состояние прототипа

- Fabric `1.21.1`, Java 21; item регистрируется и крафтится из палки, двух железных слитков и алмаза.
- ПКМ жезлом задаёт две точки box-выделения, `Shift + ПКМ` переключает отдельный блок. Первый угол box-выделения отображается жёлтым, контур текущего выделения — голубым.
- Стрелки изменяют preview сдвига относительно взгляда, `R` / `V` — по оси `Y`, `[` / `]` поворачивают группу вокруг pivot, `Enter` подтверждает операцию, `Backspace` отменяет preview, `Esc` отменяет активный preview без открытия меню, `Delete` очищает выделение. Все hotkeys переназначаемы в Minecraft Controls.
- Preview показывает допустимые позиции синим, а занятые или расположенные дальше 16 блоков — красным. Сервер повторяет проверку и атомарно применяет только допустимый сдвиг.
- Блоки, которым нужна опора (например, короткая трава, цветы и факелы), можно перенести только в позицию, где они смогут существовать после перемещения.
- Если установлен Mod Menu, кнопка Configure у MoveWand открывает настройку его биндов.
- Pivot — первая точка box-выделения либо первый добавленный отдельный блок. После успеха сервер отправляет новые координаты выделения и pivot клиенту, поэтому следующая операция работает с перемещённой группой.
- Данные `BlockEntity` переносятся через NBT. Это best effort для модовых блоков: моды, хранящие внешние ссылки на координаты, могут потребовать отдельной интеграции.

## Планируемые возможности

- Выделение прямоугольной области и отдельных блоков.
- Цветная обводка каждого выделенного блока.
- Сдвиг выделения на один блок по трём осям.
- Поворот всей выделенной структуры на 90 градусов вокруг вертикальной оси `Y`.
- Управление через hotkeys или компактный GUI со стрелками.
- Сохранение `BlockState` и данных `BlockEntity` при переносе; совместимость модовых блоков — best effort, а не обещание работы с каждым модом.

## Границы первого релиза

- Никакого копирования, удаления, замены блоков на цели или массового строительства.
- При повороте координаты всех блоков преобразуются как одна структура, а её поворачиваемые свойства (`facing`, `axis` и аналоги) меняются вместе с ней. Неподдерживаемое состояние отменяет операцию, а не оставляет сломанную постройку.
- Нет гарантий корректного переноса модов, которые хранят внешние ссылки на старые координаты; такие интеграции добавляются только после отдельной проверки.
- Первый целевой runtime — Fabric для Minecraft `1.21.1` на Java 21. Порты на другие версии начнутся после готовности и проверки ядра.
